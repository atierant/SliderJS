La nouvelle API HTML5 History (qui n'a vraiment rien à voir avec HTML - c'est une API JavaScript) permet de gérer les changements d'URL en même temps que les transitions CSS3 gèrent le glissement. Les permaliens sont toujours maintenus, le bouton de retour fonctionne comme prévu, et c'est beaucoup plus rapide que d'attendre un chargement de page complet.

Fondamentalement, on intercepte le clic, on appelle pushState() pour modifier l'URL du navigateur, on charge des données avec Ajax, puis on glisse vers elles.

Lors du clic sur le bouton de retour, un gestionnaire onpopstate est déclenché après le changement d'URL, facilitant le retour en arrière.
